## AWS Networking

공인 IP: 인터넷 사용자의 로컬 네트워크를 식별하기 위해 ISP가 제공하는 IP 주소
사설 IP: 일반 가정이나 회사 등에 할당된 네트워크 IP 주소로, 라우터에 의해 장치에 할당됨.

# Amazon VPC
논리적으로 격리된 공간을 프로비저닝하여 AWS 리소스를 구동할 수 있는 고객이 정의한 가상 네트워크
- IP 주소, 서브넷, 라우팅 규칙, 네트워크 구성, 보안 규칙 등으로 구성
- 모든 리전에 VPC 배포 가능
- 리전 내 모든 가용 영역의 리소스를 호스팅할 수 있음.
- 다중 보안 계층을 사용하여 각 서브넷에서 인스턴스에 대한 액세스 제어 가능
- 기본 할당량은 리전별 계정당 5개

서브넷? VPC를 분할하는 IP 주소의 범위로, 한 가용 영역을 선택해야 함. 서브넷을 이용해 정보를 퍼블릭과 프라이빗으로 분류할 수 있음.
IP 주소 지정? VPC 생성 후에는 주소 범위 변경 불가 -> 사이더 값을 넉넉하게 설정해야 함.

# VPC 보안
다중 보호 계층으로 인프라를 보호해야 함.
보안 그룹? AWS 리소스에 대한 인바운드 및 아웃바운드 트래픽 제어하는 방화벽
- 인바운드 및 아웃바운드 인스턴스 트래픽 제어 규칙을 설정
- 사용자 지정 보안 그룹은 허용 규칙을 지정할 수 있지만 거부 규칙을 지정할 순 없음
- 보안 그룹이 계층적으로 서로 참조하며 트래픽을 허용하는 체이닝 방식

네트워크 ACL과 비교했을 때 네트워크 ACL은 서브넷 수준의 범위에서 허용 규칙 뿐만 아니라 거부 규칙도 설정 가능하고, 번호순으로 평가됨.

# VPC로 트래픽 보내기
1. 인터넷 게이트웨이를 VPC에 연결
2. 라우팅 테이블을 인터넷 게이트웨이에 연결
3. 인스턴스에 Public IP 또는 EIP가 있는지 확인
4. nACL과 보안그룹이 관련 트래픽을 허용하는지 확인


## AWS Computing

Amazon EC2? 물리적 호스트에서 실행되는 가상 머신
- 각 인스턴스에서 게스트 운영 체제 완전히 제어 가능
- 다른 리소스로의 네트워크 연결 제공
- 필요에 따라 용량 자동 확장 또는 축소

Amazon Machine Image(AMI)?
- 루트 볼륨용 템플릿, 시작 권한, 블록 디바이스 매핑 등 제공
- 반복적으로 사용하여 효율적으로 인스턴스 시작 가능 (재사용 가능)
- 구성된 인스턴스에서 복원 가능한 백업으로서 AMI 생성 가능
